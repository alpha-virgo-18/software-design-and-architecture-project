# Online Shopping System (Software Design & Architecture)

A modular **console-based Online Shopping System** developed in **C++** as part of the **Software Design & Architecture** course. The project demonstrates the practical implementation of **Object-Oriented Programming**, **SOLID Principles**, and several **Gang of Four (GoF) Design Patterns** through a realistic e-commerce application.

---

## 📌 Project Overview

This project simulates a simple online shopping platform where:

* Customers can browse products
* Add products to their shopping cart
* Checkout using multiple payment methods
* Receive order notifications

Administrators can:

* Manage product inventory
* Configure payment methods
* Monitor system activities

The primary objective of this project is to showcase clean software architecture, modular design, and extensible implementation rather than building a full-scale commercial e-commerce platform.

## 🏗️ Architecture

The project follows a **Layered Architecture** consisting of:

* **Presentation Layer**

  * Console-based User Interface

* **Application Layer**

  * Business Logic
  * Checkout Processing
  * Cart Management

* **Data Layer**

  * In-memory product and cart storage

## 💡 SOLID Principles Applied

* Single Responsibility Principle (SRP)
* Open/Closed Principle (OCP)
* Liskov Substitution Principle (LSP)
* Interface Segregation Principle (ISP)
* Dependency Inversion Principle (DIP)
